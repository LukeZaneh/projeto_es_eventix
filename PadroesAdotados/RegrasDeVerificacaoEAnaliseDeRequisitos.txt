A nomenclatura que será utilizada para identificação dos requisitos seguirá o padrão de RFx para Requisitos Funcionais e RNFx para Requisitos Não Funcionais, nos quais o valor 'x' após a sigla refere-se à chave identificadora do requisito.
Para a organização e priorização, utilizaremos entre os requisitos as definições:
    - Essencial, para os requisitos em que o sistema não funciona caso não o tenha;
    - Importante, para os requisitos em que o sistema funciona, porém não de forma satisfatória, caso não o tenha; 
    - Desejável, requisito em que não compromete nenhuma funcionalidade básica do sistema.
Já as regras de Especificação de Requisitos que serão obedecidas são:
    - Definir um requisito por vez
    - Evitar palavras vagas e genéricas
    - Utilizar vocabulário fechado
